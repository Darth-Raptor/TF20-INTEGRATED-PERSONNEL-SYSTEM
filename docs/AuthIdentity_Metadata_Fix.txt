Task Title
Fix AuthIdentity.metadata Prisma schema/code mismatch.

Objective
Resolve the current blocking mismatch between the Prisma schema and the authentication service.

The code in src/server/auth-service.mjs reads and writes authIdentity.metadata, but the Prisma AuthIdentity model does not currently define a metadata field. This can cause Prisma validation/runtime failures once the affected auth path is executed.

Current Evidence
auth-service.mjs uses authIdentity.metadata when updating an existing Discord identity:

metadata: guildPayload ?? authIdentity.metadata ?? {},

It also writes metadata when creating a new Discord identity:

metadata: guildPayload ?? {},

These usages are in the Discord account resolution flow.

The Prisma AuthIdentity model includes fields for provider, provider account ID, username, display name, guild verification, link/unlink state, and primary identity state, but the inspected schema section does not include metadata.

Required Fix
Add a nullable JSON metadata field to the AuthIdentity Prisma model.

Recommended schema addition:

metadata Json?

The field should be added to the AuthIdentity model in prisma/schema.prisma.

Expected Model Shape
The relevant model should end up conceptually like this:

model AuthIdentity {
  id                      String    @id @default(cuid())
  accountId               String
  account                 Account   @relation(fields: [accountId], references: [id])
  provider                String
  providerAccountId       String
  username                String?
  displayName             String?
  lastGuildVerifiedAt     DateTime?
  guildMembershipRequired Boolean   @default(true)
  metadata                Json?
  linkedAt                DateTime  @default(now())
  unlinkedAt              DateTime?
  isPrimary               Boolean   @default(false)

  @@unique([provider, providerAccountId])
  @@index([accountId])
}

Do not rename the existing field usages in auth-service.mjs unless there is a strong reason. The cleanest fix is to add the missing schema field.

Migration Requirement
Create a Prisma migration for the schema change.

Expected migration outcome:

ALTER TABLE `AuthIdentity` ADD COLUMN `metadata` JSON NULL;

Exact SQL may vary depending on Prisma/MySQL output.

At minimum, confirm these flows do not throw Prisma unknown-field errors:

1. Existing Discord identity login:

The current auth service creates pending accounts and initial Discord identities in a transaction, assigns the pending role, and writes an audit entry.
